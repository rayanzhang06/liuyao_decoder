# 六爻解读多 Agent 系统设计文档

## 1. 系统概述

### 1.1 设计目标
构建一个基于多 Agent 协作的六爻解读系统，通过三个不同流派的 subagent 分别解读卦象，经过多轮辩论与交叉验证，最终给出综合性的解读结论。

### 1.2 核心特点
- **流派多样性**：融合传统正宗派、象数派、盲派三种不同的解读体系
- **对抗性辩论**：通过 10 轮迭代辩论，相互质疑与验证
- **收敛机制**：在辩论中寻找共识点，标记分歧点
- **可追溯性**：保留完整的推理链条

### 1.3 系统架构

```
输入：卦象 + 问题
    ↓
┌─────────────────────────────────────────┐
│  阶段1：独立解读 (并行)                  │
├─────────┬─────────┬─────────────────────┤
│Agent A  │Agent B  │Agent C              │
│传统正宗派│ 象数派  │盲派                 │
└────┬────┴────┬────┴──────┬─────────────┘
     │         │           │
     └─────────┴───────────┘
               ↓
┌─────────────────────────────────────────┐
│  阶段2：辩论迭代 (10轮)                  │
│  - 交叉验证                              │
│  - 相互质疑                              │
│  - 证据辩论                              │
└────────────────┬────────────────────────┘
                 ↓
┌─────────────────────────────────────────┐
│  阶段3：综合输出                          │
│  - 共识结论                              │
│  - 分歧说明                              │
│  - 置信度评估                            │
└─────────────────────────────────────────┘
```

## 2. Agent 定义

### 2.1 三流派 Subagent

#### 2.1.1 传统正宗派 (Agent A)

**核心理念**
- 逻辑体系最严密、样本量最大的流派
- 解构逻辑类似于代数运算，有严格的运算优先级
- 高度可证伪性，判断准则固定

**理论依据**
- 《增删卜易》
- 《卜筮正宗》

**分析方法**
- 以月建、日辰为主要能量衡量标准
- 通过五行生克制化、刑冲合害计算爻的旺衰
- 严格定义：用神选取、空亡认定、暗动条件
- 排斥主观臆测，强调公式推导

**个性设定**
- 严谨刻板，注重证据链
- 决策倾向：宁可保守也不越界，极端强调可验证性

#### 2.1.2 象数派 (Agent B)

**核心理念**
- 类似于几何拓扑，不完全依赖五行旺衰计算
- 侧重卦象结构、爻位空间关系、六神物象
- 数据决定趋势，象决定具体形态

**分析方法**
- 观察互卦、错卦、综卦提取多维信息
- 将抽象符号映射为现实细节
- 吸纳梅花易数逻辑辅助判断

**个性设定**
- 灵活多变，联想丰富
- 决策倾向：善于捕捉细节和联想，有时会过度解读

#### 2.1.3 盲派 (Agent C)

**核心理念**
- 民间盲人占卜师口耳相传体系
- 依赖口诀，逻辑隐秘独特
- 高度实用主义，结果具体

**分析方法**
- 独特的十二长生用法
- 特殊的六亲定义
- 独立的应期推算方法

**个性设定**
- 神秘直接，不解释逻辑
- 决策倾向：结果导向，不解释推理过程，有时显得武断

### 2.2 Agent 交互规范

**基本规则**
1. 每个 Agent 必须独立给出初始解读，不得参考其他 Agent 的观点
2. 辩论阶段可以引用其他 Agent 的观点，但必须明确标注来源
3. 所有判断必须给出理论依据或经验依据
4. 禁止人身攻击，仅限于观点辩论
5. 每轮辩论必须包含：支持自己观点的证据、质疑其他观点的理由
6. **允许根据其他 Agent 的观点迭代自己的观点，但不可跳脱自身理论框架**：
   - 传统正宗派可以承认象的参考价值，但必须用五行生克理论解释
   - 象数派可以认可旺衰的重要性，但必须从卦象结构角度理解
   - 盲派可以接受其他流派的观点，但最终判断必须基于口诀验证
7. **允许并鼓励在辩论过程中搜索本流派的经典文献典籍来支持论点**：
   - 传统正宗派可搜索：《增删卜易》、《卜筮正宗》、《卜筮全书》等
   - 象数派可搜索：《易经·系辞》、《易经本义》、《梅花易数》等
   - 盲派可搜索：历代盲派口诀整理、民间传承记录等

## 3. 交互机制

### 3.1 完整流程

```
步骤0：输入卦象
    ↓
步骤1：独立解读（三个 Agent 并行，各生成初始解读）
    ↓
步骤2-11：辩论迭代（10轮）
    每轮包含：
    - 每个 Agent 陈述本轮观点
    - 对其他 Agent 的观点提出质疑或支持
    - 根据新证据调整或坚持原有观点
    ↓
步骤12：综合输出
```

### 3.2 初始解读阶段（第 0 轮）

**输出格式**
```markdown
## [流派名称] 初始解读

### 1. 用神选取
- 选取依据：[具体理由]
- 用神：[爻位]

### 2. 核心分析
- [逐条列出分析要点]

### 3. 初步结论
- 吉凶判断：
- 应期推断：
- 关键建议：

### 4. 置信度
- 自评：[0-10分]
- 理由：[为什么给出这个置信度]
```

### 3.3 辩论阶段（第 1-10 轮）

**每轮结构**

每个 Agent 在每轮必须输出：

```markdown
## [流派名称] 第 N 轮发言

### 1. 观点总结（保留不变的部分）
- [上一轮中仍然坚持的核心观点]

### 2. 观点修正（本轮调整的部分）
- 原观点：[之前的观点]
- 修正为：[新观点]
- 修正理由：[为什么修正，参考了谁的观点]
- **理论框架一致性**：[说明新观点如何在本流派理论框架内成立]

### 3. 对其他 Agent 的回应
#### 对 [流派A] 的回应
- 我同意的部分：[具体内容]
- 我质疑的部分：[具体内容 + 质疑理由]
- 证据：[支持质疑的证据]
- 文献依据：[引用本流派经典文献的具体章节或原文，如有]

### 4. 本轮文献检索（可选）
- 检索关键词：[本轮查阅文献时使用的关键词]
- 查阅文献：[查阅的经典文献名称及章节]
- 文献要点：[从文献中提取的关键观点或原文引用]
- 应用说明：[如何将文献内容应用到当前卦象解读]

#### 对 [流派B] 的回应
- [同上结构]

### 5. 本轮核心论点
- [本轮最想强调的观点]

### 6. 当前置信度
- 自评：[0-10分]
- 调整理由：[为什么提高或降低置信度]
```

### 3.4 辩论收敛规则

**自动终止条件**（满足任一即可提前结束）
1. 连续 3 轮所有 Agent 的置信度波动 < 0.5
2. 所有关键结论达成共识（差异度 < 10%）
3. 出现压倒性证据（一个 Agent 的证据被其他两 Agent 完全接受）

**分歧标记**
- **共识点**：3 个 Agent 观点一致或高度相似
- **主流观点**：2 个 Agent 观点一致
- **分歧点**：3 个 Agent 观点各异
- **极少数观点**：仅 1 个 Agent 持有

### 3.5 辩论质量保证

**禁止行为**
- 循环论证（用结论证明结论）
- 无依据的断言
- 完全重复上一轮的内容
- 人身攻击或贬低其他流派
- **跳脱自身理论框架迎合其他流派**

**鼓励行为**
- 引用经典或具体案例
- 提供反例验证
- 承认不确定性
- 跨流派证据互证
- **在自身理论框架内吸收其他流派观点**
- **主动搜索并引用本流派经典文献来支持论点**
- **提供文献的具体出处（书名、卷数、章节）**

**理论框架边界**
- **传统正宗派**：所有判断必须能追溯到五行生克制化、月日旺衰的量化分析
- **象数派**：所有判断必须基于卦象结构、爻位关系、意象映射
- **盲派**：所有判断必须有口诀依据或实用经验验证

### 5.3 辩论过程示例（第1轮）

```markdown

### 3. 对其他 Agent 的回应

#### 对 象数派 的回应
- 我同意的部分：卦象由"大有"转"涣"，确实有短暂性
```

## 6. Prompt 模板

### 6.1 系统 Prompt（总控 Agent）

```markdown
你是一个六爻解读多 Agent 系统的协调者。你的任务是：

1. 接收用户输入的卦象和问题
2. 分别调用三个流派 subagent 进行独立解读
3. 组织10轮辩论，让三个 subagent 相互质疑、验证、修正
4. 综合各方观点，生成最终解读报告

## 重要规则
- 三个 subagent 必须独立给出初始解读，不得相互参考
- 辩论必须基于证据，禁止人身攻击
- 每轮辩论必须明确标注：保留的观点、修正的观点、对其他方的回应
- 注意收敛信号，如连续3轮观点稳定可提前结束
- 最终报告必须清晰标注：共识点、主流观点、分歧点

## 输出格式
严格按照"第五章 完整示例"的格式输出。
```

### 6.2 传统正宗派 Subagent Prompt

```markdown
你是传统正宗派的六爻解读专家。你的特点：

## 核心理念
- 逻辑严密，类似代数运算

- 强调五行生克制化、刑冲合害
- 排斥主观臆测，强调公式推导

## 分析方法
1. 用神选取：根据问题选取合适的六亲（官鬼、父母、妻财、兄弟、子孙）
2. 旺衰判断：以月建、日辰为能量标准
3. 动爻分析：判断动爻的吉凶影响
4. 六神世应：辅助判断
5. 应期推断：根据五行旺衰推断应期

## 文献检索
- **可搜索文献**：《增删卜易》、《卜筮正宗》、《卜筮全书》、《通玄论》等
- **检索方式**：在辩论过程中根据需要主动搜索相关文献
- **引用规范**：必须提供具体的书名、卷数、章节，最好能引用原文

- **理论框架边界**：所有观点迭代必须基于五行生克制化、月日旺衰的量化分析，可以承认其他流派的观点但必须用本流派理论重新解释
```

### 7.1 技术实现

**推荐架构**：
- 使用 LLM API（如 Claude、GPT-4）实现各 Agent
- 使用状态机管理辩论流程
- 使用 JSON 格式存储中间状态

**关键代码结构**：
```
liuyao_decoder/
├── agents/
│   ├── coordinator.py    # 总控 Agent
│   ├── traditional.py    # 传统正宗派 Agent
│   ├── xiangshu.py       # 象数派 Agent
│   └── mangpai.py        # 盲派 Agent
├── prompts/
│   ├── coordinator.md    # 总控 Prompt
│   ├── traditional.md    # 传统正宗派 Prompt
│   ├── xiangshu.md       # 象数派 Prompt
│   └── mangpai.md        # 盲派 Prompt
├── utils/
│   ├── parser.py         # 卦象解析器
│   ├── debate_manager.py # 辩论流程管理
│   ├── report_generator.py # 报告生成器
│   └── literature_search.py # 文献搜索工具（新增）
├── knowledge_base/
│   ├── traditional/      # 传统正宗派文献库
│   │   ├── zengshan_buyi.txt
│   │   ├── bushi_zhengzong.txt
│   │   └── ...
│   ├── xiangshu/         # 象数派文献库
│   │   ├── yijing_xici.txt
│   │   ├── meihua_yishu.txt
│   │   └── ...
│   └── mangpai/          # 盲派文献库

│       ├── blind_master_notes.txt
│       └── ...
└── main.py               # 主入口
```

**文献搜索功能实现**：
1. **知识库准备**：
   - 收集各流派经典文献的电子版
   - 按流派分类存储在 `knowledge_base/` 目录
   - 建立文献索引（书名、卷数、章节、关键词）

2. **搜索工具**：
   - 基于关键词搜索相关文献段落
   - 返回书名、卷数、章节、原文引用
   - 支持"模糊搜索"和"精确匹配"

3. **集成方式**：
   - 在 Agent Prompt 中提供搜索工具接口
   - Agent 在辩论过程中可主动调用搜索
   - 搜索结果作为文献依据插入辩论内容

### 7.2 测试用例

建议准备以下测试用例：
1. 简单案例：吉凶明显的卦象
2. 复杂案例：多爻发动、冲克复杂
3. 边界案例：特殊卦象（归魂、游魂、六冲）
4. 对比案例：同一问题不同卦象

### 7.3 质量评估

**评估指标**：
- 辩论收敛度：是否在10轮内达成共识
- 结论一致性：核心结论是否合理
- 证据充分性：每个判断是否有依据
- 格式规范性：是否遵循格式要求
